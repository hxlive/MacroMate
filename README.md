## 📖 项目简介

**宏助手 (Macro Assistant)** 是一款开源免费的**桌面自动化工具**，专为简化重复性任务而设计。无论是游戏自动化、办公流程优化，还是 UI 测试，宏助手通过**图像识别**和 **OCR 文字识别**，帮助用户快速录制和执行自动化脚本。

无需编程经验，通过直观的 GUI 界面即可轻松上手。

项目在性能、响应速度和交互设计上进行了多轮深度优化，提供毫秒级图像查找、即时热键中断和极简的现代 UI，适合个人用户和开发者高效使用。

## ⚡ 核心功能与亮点

* **🛡️ 即时安全中断 (v1.6.2+)**：支持通过全局快捷键 (`Ctrl+F11`) 或悬浮状态条一键**立即停止**正在运行的宏，基于底层的线程异常注入机制实现，响应优先级最高，按下即停。
* **🔀 步骤屏蔽与启用**：支持右键点击步骤序列中的任意步骤，通过菜单一键“屏蔽”或“启用”，方便对特定逻辑进行调试（控制流步骤除外）。
* **🚀 极速启动与防卡死**：采用异步加载机制。重型 UI 初始化延迟到主循环启动后执行，且 OCR 引擎的检测与预加载移至后台线程，彻底解决打包后冷启动界面“未响应”的问题。
* **🔍 匹配增强模式**：可选开启“增强模式”，图像查找自动启用多级缩放匹配（`1.0, 0.9, 1.1, 0.8, 1.2`），文字识别（OCR）自动进行 2 倍缩放，大幅提升低分辨率或多变场景下的识别率。
* **💼 多引擎 OCR 智能集成**：内置 `WinOCR`（Windows 10/11 内置，响应最快）、`RapidOCR`（基于 ONNX 深度学习，识别最准）和 `Tesseract`（本地离线兜底），可根据系统环境自动推荐或手动指定。
* **📁 流程控制与安全执行 (RUN)**：支持完整的 `IF/ELSE` 判断、`LOOP` 循环（支持基于图像/文本条件的直到循环）。新增安全沙箱控制下的 `RUN` 步骤，可在授权下执行外部脚本或程序。
* **⌨️ 全局热键与防冲突**：自动检测 `Ctrl+F10` 启动和 `Ctrl+F11` 停止快捷键是否被其他系统程序占用。支持在设置中自定义热键。
* **🖥️ 悬浮迷你状态栏**：运行宏时可自动最小化主界面，并在屏幕顶部居中显示精美的迷你状态栏，显示当前步骤，并支持快捷键和点击停止。
* **🖱️ 辅助定位与即时测试**：
    * **实时坐标追踪**：添加定位步骤时，鼠标移动即可实时抓取并显示绝对坐标。
    * **即时功能测试**：无需运行整个宏，在添加“查找图像/文本/AI”步骤前可单步即时测试，验证匹配结果与坐标。

## 📚 动作列表详解

| 序号 | 动作 | 功能描述 |
| :--- | :--- | :--- |
| **01** | **查找图像** | (核心) 在全屏或自定义区域内查找指定图像，找到后将鼠标移动到其中心。支持置信度调节。 |
| **02** | **查找文本 (OCR)** | (核心) 在全屏或区域内识别并定位指定文字。支持将识别结果保存到剪贴板，并使用正则表达式进行二次提取。 |
| **03** | **相对移动** | 从 *上一个动作* 执行时的鼠标位置，按指定的偏移量移动鼠标。 |
| **04** | **移动到 (绝对坐标)** | 将鼠标移动到屏幕的精确 X, Y 坐标 (提供实时鼠标坐标参考)。 |
| **05** | **点击鼠标** | 在当前鼠标位置执行点击 (支持左键/右键/中键，以及单击/双击)。 |
| **06** | **滚动滚轮** | 在当前或指定位置滚动鼠标滚轮（支持向上或向下滚动）。 |
| **07** | **等待** | 暂停宏，等待固定时间 (毫秒)。 |
| **08** | **输入文本** | 模拟键盘文本输入（支持直接粘贴，支持中文与特殊字符）。 |
| **09** | **按下按键** | 模拟按下单个按键或组合快捷键 (如 `enter`, `ctrl+c`, `ctrl+v`)。 |
| **10** | **AI 自然语言指令** | 利用 VLM 视觉语言模型大模型，分析屏幕画面并根据指令自动定位。 |
| **11** | **激活窗口 (按标题)** | 查找标题匹配的窗口，并将其激活并置顶到最前端。 |
| **12** | **备注** | 为步骤添加说明性备注，仅用于逻辑标记和阅读，不执行实际动作。 |
| **13** | **IF 找到图像** | (流程控制) 如果在指定范围内找到该图像，则执行 `IF` 到 `END_IF` 之间的步骤。 |
| **14** | **IF 找到文本** | (流程控制) 如果在指定范围内通过 OCR 匹配到对应文本，则执行对应步骤（支持正则匹配）。 |
| **15** | **ELSE** | (流程控制) 与 `IF` 配对，当 `IF` 条件不满足时执行 `ELSE` 块中的步骤。 |
| **16** | **END_IF** | (流程控制) 标记 `IF` 判定块的结束。 |
| **17** | **执行命令/脚本/文件** | (系统动作) 运行系统命令或启动本地的脚本和文件（需在主界面开启“启用 RUN 步骤”）。 |
| **18** | **循环开始 (Loop)** | (流程控制) 标记循环体起点。支持无限循环或设置退出条件（直到图像消失/文字出现）。 |
| **19** | **结束循环 (EndLoop)** | (流程控制) 标记循环体的结束。

## 💡 特色高级功能使用指南

### 1. 执行外部命令/脚本/文件 (RUN) 

为了防范系统安全风险，该功能默认关闭。您需要在主界面左下角勾选 **“启用 RUN 步骤”** 才能生效。它提供以下三种强大的工作模式，能够将宏助手与您的本地脚本和系统指令无缝联动：

#### ① 文件写入模式 (`file`)
* **功能描述**：将指定的文字内容写入或以追加方式保存到本地的任意文件中（若目标文件夹不存在，程序会自动创建）。
* **应用场景**：自动把 OCR 提取到的订单号、验证码、计算结果或时间，保存到本地的 `.txt` 或 `.csv` 表格中。
* **动态占位符支持**：在写入的内容中，您可以使用以下占位符：
  * `{CLIPBOARD}`：自动替换为当前剪贴板的内容（或前几步 OCR 抓取并提取的内容）。
  * `{DATETIME}`：自动替换为当前的系统日期和时间（格式为 `yyyy-MM-dd HH:mm:ss`）。

#### ② 系统命令执行模式 (`command`)
* **功能描述**：在系统后台直接执行 CMD 命令行指令（以 Shell 模式运行，支持管道与重定向）。
* **常用参数**：
  * **命令**：执行的程序或指令（如 `ping`、`taskkill`、`explorer.exe`）。
  * **参数**：传递给命令的后缀参数（如 `baidu.com -n 3`）。
  * **保存输出到剪贴板**：如果勾选，该命令执行完毕后在控制台输出的内容（Stdout/Stderr）会被自动复制到剪贴板，供后续步骤（如“输入文本”）直接粘贴使用。
* **典型示例**：
  * 启动本地软件：命令填写 `explorer.exe`，参数填写 `"C:\Program Files\YourApp\app.exe"`
  * 强制关闭卡死的浏览器：命令填写 `taskkill`，参数填写 `/f /im chrome.exe`

#### ③ 本地脚本执行模式 (`script`)
* **功能描述**：使用特定的解释器在后台调用并运行您写好的本地脚本代码文件。
* **常用参数**：
  * **脚本路径**：您脚本文件在本地的绝对路径（如 `D:\scripts\parse_data.py`）。
  * **解释器**：下拉选择执行该脚本的环境，支持 `python`、`node` (JS)、`powershell` 或 `cmd` (批处理)。
  * **脚本参数**：脚本执行时所需的外部参数。

### 2. 文本识别 (OCR) 正则提取与数据链传递

在“02. 查找文本 (OCR)”或“14. IF 找到文本”步骤中，您可以开启 **“保存识别结果到剪贴板”**。开启后会展开高级提取配置，允许您从满屏文字中精准提炼所需的数据：

#### ⚙️ 正则表达式提取模式 (Regex Extract)
OCR 识别出来的往往是一整行或大段文字，配合 **“提取模式（正则）”** 可以实现精准字段截取。
* **提取纯数字验证码**：
  * 正则表达式：`\d+`
  * 示例：若屏幕上识别出的原始文字为 `[宏助手] 您的验证码是 4829，请在5分钟内输入`。提取后，剪贴板只保留 `4829`。
* **提取特定前缀后面的金额/订单号**：
  * 正则表达式：`订单号: (\d+)` （使用小括号 `()` 构建捕获组，程序会自动提取括号内匹配的文字）
  * 示例：若屏幕识别出 `购买成功，订单号: 202605210087`。提取后，剪贴板只保留 `202605210087`。

#### 🔄 跨步骤“数据链”联动案例
通过将 OCR 提取与 RUN 动作结合，您可以轻松实现自动化数据流：
1. **步骤一（OCR 识别）**：使用 `查找文本 (OCR)` 搜索屏幕上的快递单号，勾选“保存识别结果到剪贴板”，正则填入 `单号: ([A-Za-z0-9]+)`。
2. **步骤二（记录到本地）**：使用 `执行命令/脚本/文件 (RUN)` 选择 `file` 模式，写入文件 `D:\log.txt`，勾选“追加模式”，内容填入：`[{DATETIME}] 提取单号: {CLIPBOARD}\n`。
3. **步骤三（自动查单）**：使用 `激活窗口` 切到浏览器，并在搜索框使用 `输入文本`，内容填写 `{CLIPBOARD}` 并模拟按下 `Enter` 键。

## 🛠️ 安装与依赖

### 1. 库依赖

项目依赖以下 Python 第三方库。可以直接运行 `pip install -r requirements.txt` 安装：

```text
ttkbootstrap
pyautogui
pynput
pygetwindow
opencv-python
Pillow
pytesseract
rapidocr-onnxruntime
windows-ocr
```

### 2. OCR 引擎配置说明

* **WinOCR** (推荐, 速度最快):
    * 仅支持 Windows 10 及以上系统。
    * 无需任何配置，直接可用，响应速度极快。
* **RapidOCR** (推荐, 精度最高):
    * 运行 `pip install rapidocr-onnxruntime` 自动安装。
    * **重要提示**：如果在启动时或运行时提示 `DLL load failed`，表明您的系统缺少 VC++ 运行库。您必须安装最新的 [Microsoft Visual C++ Redistributable](https://learn.microsoft.com/zh-cn/cpp/windows/latest-supported-vc-redist) 以保证 ONNXRuntime 能正常运行。
* **Tesseract**:
    * 作为本地离线兜底引擎。
    * **方式一 (免配置)**: 解压 `tesseract_local.7z`，确保解压出的 `tesseract_local` 文件夹（内含 `tesseract.exe` 和 `tessdata` 语言包）与主脚本 `MacroAssistant.py` 放在同一目录下即可。
    * **方式二 (全局安装)**: 安装 Tesseract 客户端，并将 `tesseract.exe` 路径添加到系统 `PATH` 环境变量中。

## ⚠️ 注意事项与安全性

* **🔒 完全离线与数据安全**：本工具默认完全离线运行，无任何隐式网络请求，您的宏配置文件、截图和输入数据全部保存在本地。
* **📜 宽松的开源协议**：本项目基于 **MIT 协议** 开源，无论是商业使用还是私人修改均可自由进行。
* **📊 性能监控与调试**：
    * 每次运行宏时，控制台都会打印详细日志，并在执行完毕后在状态栏输出匹配率和耗时。
    * 可在项目目录下查看 `macro_perf.log` 获取底层性能分析数据。
* **📦 关于 exe 打包运行**：
    * 如果使用 PyInstaller 将项目打包为单文件/文件夹 exe，确保 `Icon_Res` 图标和配置资源包含在内。
    * 打包后如提示 RapidOCR 不可用，请检查打包 spec 文件是否已使用 `collect_all` 正确收集 `rapidocr_onnxruntime` 的 binaries 和 datas。
